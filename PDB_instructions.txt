----REQUIRED INSTALLATIONS-------
1. Install MySQL 5.5 - user: root pwd: enter a suitable password (including Workbench if possible), check installation path (!)
2. Install .NET 4.0 Client Profile and Reboot
3. Install .NET Framework 4.0 Full
4. Install MS Visual C++ Redistributable
5. Install mysql-connector-net-6.5.4
6. Install MySQL Workbench if you haven yet (if in doubt, just accept all default settings)

----SETTING UP PDB-------
7. Download and copy @Arma2Net folder to ArmA2(OA) Root directory (see attached zip file) - https://dev-heaven.net/attachments/19142/@Arma2NET.7z
8. Check that MYSQL Server is running and import MSO db using MySQL Workbench (get MSO[date].sql from mission folder \core\modules\persistentDB\ or below in attachments)
9. In MySQL Workbench > Manage Security > Setup a user called arma (set password) and give them dba permissions, ensure user can only connect from localhost.
10. Update Server launch shortcut and add @Arma2NET to mod line
11. Copy Databases.txt (from Arma2netMysql folder) to (C:\Users\%USERNAME%\AppData\Local\Arma2MySQL) folder, open the file, delete the #, change username to arma and change password (last two entries on line)
12. Launch the server shortcut, start up a mission with PDB and make sure it is switched to "ON" on params.

----DEBUGGING-------
13. Install baretail log monitor
14. In baretail open the server rpt and the arma2net log file (C:\Users\%USERNAME%\AppData\Local\Arma2NET) - will indicate if MySQLPlugIn has been successfully loaded.
15. Once in game, in baretail open the mysqlplugin log (C:\Users\%USERNAME%\AppData\Local\Arma2MySQL\logs\[date].log - will indicate if SQL calls have been successful.

NOTE: Make sure in your locale regional settings the decimal seperator (maybe a comma ,) is set to dot (.)!
